Clean Code & Qualimétrie avec SonarQube

Qu'est-ce que le Clean Code ?

  • Un code facile à lire et à comprendre
  • Un code facile à maintenir et à faire évoluer
  • Un code qui respecte les bonnes pratiques de développement

Principes

Conventions de codage

  • naming conventions
  • code formatting
  • commenting

Revues de code

  • Une pratique de développement collaborative
  • La vérification de la qualité du code par les pairs
  • L'amélioration continue du code

Analyse de code

  • Une pratique de développement automatisée
  • La vérification de la qualité du code par des outils d'analyse statique
  • L'amélioration continue du code grâce à des rapports d'analyse

debt / dette technique

  • Un concept qui décrit les compromis faits dans le développement du code
  • Peut être intentionnelle ou non intentionnelle
  • Peut être payée plus tard avec des intérêts

Outils

SonarQube

  • Un outil d'analyse statique du code
  • La détection des problèmes de qualité et de sécurité
  • L'amélioration continue de la qualité du code

Déploiement de sonarQube avec Docker


	docker run -d --name sonarqube -p 9000:9000 sonarqube:community
